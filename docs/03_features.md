# WorkPilot Features (MVP)

## Overview

This document defines the core features of WorkPilot Version 1.0.

The goal is to build a minimal but usable AI Receptionist system.

---

## 1. AI Chat Receptionist

- Receive messages from residents
- Understand basic intent (question / request / complaint)
- Respond using AI + knowledge base

---

## 2. FAQ System

- Store common questions
- Provide instant answers
- Reduce repetitive staff workload

Example:
- Opening hours
- Fees
- Contact information

---

## 3. Maintenance Ticket System

- Create ticket from resident message
- Assign category (electricity, water, repair)
- Track status (open / in progress / done)

---

## 4. Admin Dashboard

- View all messages
- View all tickets
- Update ticket status
- Monitor system usage

---

## 5. Notification System

- Notify staff via LINE or email
- New ticket alerts
- Status update alerts

---

## 6. Knowledge Base

- Upload documents (PDF, rules, manuals)
- AI uses data to answer questions

---

## 7. Multi-User Support (Basic)

- Separate data per building/company
- Each client has isolated system

---

## OUT OF SCOPE (IMPORTANT)

These features are NOT included in MVP:

- Payment system automation
- AI learning by itself
- Mobile app
- IoT / hardware control
- Advanced analytics
- Fully autonomous AI decision making

---

## Core Principle

If a feature does not help:

- Answer residents faster
- Reduce staff workload
- Create tickets automatically

→ It is NOT included in MVP
